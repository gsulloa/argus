#### Unreleased

- Plain Enter now applies filters in the data grid across Postgres, MySQL, MSSQL, and Dynamo (fixes #53).
- Add MySQL / MariaDB support reaching feature parity with Postgres for connection management, schema browser, data grid, inline editing, SQL editor, table structure viewer, and bulk columns cache.
- Note: `TINYINT(1)` is decoded as boolean (MySQL convention). Multi-statement runs that include `CREATE PROCEDURE / FUNCTION / TRIGGER / EVENT` are rejected; run them as single statements.
- Add Microsoft SQL Server support (SQL Server 2017+, Azure SQL Database, Azure SQL Managed Instance) with the same full feature set: connection management, schema browser with sys.schemas / sys.databases browsing, virtualized data grid with inline editing, SQL editor with `GO` batch support, and table structure viewer.
- URL parser accepts `mssql://`, `sqlserver://`, `microsoftsqlserver://` (TablePlus export form), `jdbc:sqlserver://`, and ADO.NET `Key=Value;` connection strings. TablePlus-specific `tLSMode` query param is honored (`0` → no TLS, `1` → TLS with trust-cert, `2` → strict TLS); other vendor extras (`statusColor`, `safeModeLevel`, `driverVersion`, …) are silently ignored so paste-from-TablePlus just works.
- v1 limitations for MS SQL Server:
  - SQL Authentication only (Windows Authentication and Azure AD / Entra ID are not supported).
  - `Encrypt=true` is the default; set Encrypt to Off only for local dev or trusted private networks.
  - `GO` separator is required before `CREATE PROCEDURE / FUNCTION / TRIGGER / VIEW` when preceded by other statements in the same SQL editor buffer.
  - Edit refresh on tables with `INSTEAD OF` triggers falls back transparently from the `OUTPUT`-clause path to a re-fetch via `SELECT WHERE pk = ?`; slightly slower but always correct.
  - `BIT` is the canonical boolean type (unlike MySQL where `TINYINT(1)` maps to bool).
  - `GEOMETRY`, `GEOGRAPHY`, `HIERARCHYID`, `SQL_VARIANT`, and `ROWVERSION` columns are read-only in the data grid; use the SQL editor to modify them.
  - `IDENTITY` columns cannot be inserted via the data grid (values are auto-assigned by the server); use `SET IDENTITY_INSERT ON` in the SQL editor for manual inserts.
  - The binary now includes two `rustls` versions (0.21 from `sqlx`/tiberius and 0.23 from `tokio-postgres-rustls`) — same as the MySQL baseline; no change to the trust anchor behaviour.
